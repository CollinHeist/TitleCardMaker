v2.0-alpha.11.0
Released July 21, 2024

Major Changes

Begin implementing automated front-end testing with Cypress
Allow importing series poster, series backgrounds, and season posters via Kometa / MediUX YAML (this works for Emby, Jellyfin, and Plex)
Add ability to delete all Series which are not a part of any Sync
Adds a global setting called "Delete Un-Synced Series" on the Settings page
If enabled, TCM will delete the Series, Cards, and Source Images (if enabled) of any Series which is not included in any Sync
This is a potentially destructive setting - be careful before enabling. Deleted Cards and Source Images are not backed up by TCM
Add global option for the ImageMagick executable path
This option replaces the TCM_IM_PATH environment variable
This setting is only applicable to some Windows users
Significantly improve front-end error messages for Title Card validation failures - TCM will now display more "human readable" messages of the specific field validations which caused the creation to fail
Reorganize the Templates page

Various security improvements

Encrypt all private Connection details (URLs and API keys) within the database, rather than keep them in plaintext

Properly redact secrets which may appear in exception tracebacks - this most commonly occurred in Plex API errors which include the X-Plex-Token as a header parameter
Redact secrets in order of descending length to prevent possible "leaking" partial substrings of the secret if contained in another secret - e.g. if you had an API key of ABC and ABCDEF, previously ABC could have been redacted first , resulting in DEF kept as-is in the logs
